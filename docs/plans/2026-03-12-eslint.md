# ESLint Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 为前端和 `server/index.js` 补齐 ESLint 配置，并让 `npm run lint` 在当前仓库上通过。

**Architecture:** 使用 ESLint 8 传统配置文件 `.eslintrc.cjs` 和 `.eslintignore`，保持与现有依赖兼容。通过更新 lint 脚本把服务器 JS 一并纳入，再根据实际 lint 输出修复仓库中的当前违例。

**Tech Stack:** ESLint 8, TypeScript ESLint, React Hooks plugin, React Refresh plugin, Vite

---

### Task 1: Establish the failing lint baseline

**Files:**
- Modify: `package.json`
- Create: `.eslintrc.cjs`
- Create: `.eslintignore`

**Step 1: Verify the current failure**

Run: `npm run lint`

Expected: fail because no ESLint configuration file exists.

**Step 2: Add the minimal configuration**

Create `.eslintrc.cjs` and `.eslintignore`, update `package.json` so lint includes `js,cjs,ts,tsx`.

**Step 3: Run lint to get the first real rule violations**

Run: `npm run lint`

Expected: fail on code issues rather than configuration lookup.

### Task 2: Fix lint violations in source files

**Files:**
- Modify: source files reported by ESLint

**Step 1: Fix violations one cluster at a time**

Prioritize real issues first, then unused vars/imports.

**Step 2: Re-run lint after each cluster**

Run: `npm run lint`

Expected: error count decreases until zero.
