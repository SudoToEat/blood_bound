# 血契猎杀 - 快速使用指南

## 🚀 一键启动

```bash
# 克隆项目后，直接运行
./start.sh
```

## 📋 使用步骤

### 1. 主持人端

1. **访问游戏**
   - 打开浏览器访问: http://localhost:5173
   - 或网络访问: http://192.168.5.115:5173

2. **创建房间**
   - 点击"创建房间"
   - 选择玩家数量 (6-12人)
   - 点击"创建房间"按钮
   - 获得房间号和二维码

3. **分享给玩家**
   - 让玩家扫描二维码
   - 或分享访问链接给玩家

### 2. 玩家端

1. **加入游戏**
   - 扫描主持人提供的二维码
   - 或直接访问链接: `/access/[房间号]/[玩家ID]`
   - 例如: `http://192.168.5.115:5173/access/123456/1`

2. **查看身份**
   - 成功加入后自动显示个人身份
   - 实时同步游戏状态

## 🔧 常用命令

### 启动服务
```bash
# 一键启动 (推荐)
./start.sh

# 分别启动
npm run server        # 后端服务
npm run dev -- --host # 前端服务

# 同时启动
npm run dev:full
```

### 检查状态
```bash
# 检查服务状态
./scripts/check-status.sh

# 测试功能
./scripts/test-multi-browser.sh
```

### 停止服务
```bash
# 如果使用 start.sh 启动，按 Ctrl+C
# 或者手动停止进程
pkill -f "node server/index.js"
pkill -f "vite"
```

## 🌐 访问地址

- **本地访问**: http://localhost:5173
- **网络访问**: http://192.168.5.115:5173
- **后端API**: http://localhost:3000

## 🎮 游戏流程

1. **准备阶段**
   - 主持人创建房间
   - 玩家扫描二维码加入
   - 确认所有玩家就绪

2. **游戏进行**
   - 玩家查看个人身份
   - 线下进行游戏互动
   - 实时同步游戏状态

3. **结束游戏**
   - 游戏结束后关闭浏览器
   - 或使用 Ctrl+C 停止服务

## 🐛 常见问题

### 无法访问游戏
```bash
# 检查服务状态
./scripts/check-status.sh

# 重启服务
./start.sh
```

### 玩家无法加入
- 确认房间号正确
- 检查网络连接
- 确保所有设备在同一网络

### 实时同步不工作
- 检查 WebSocket 连接
- 查看浏览器控制台
- 确认服务器正常运行

## 📱 移动端使用

- 支持手机浏览器访问
- 响应式设计，自动适配屏幕
- 建议使用 Chrome 或 Safari

## 🔒 安全提示

- 仅在可信任的局域网内使用
- 不要暴露到公网（除非有安全措施）
- 游戏结束后及时关闭服务

## 📞 技术支持

如果遇到问题：
1. 运行 `./scripts/check-status.sh` 检查状态
2. 查看浏览器控制台错误信息
3. 检查网络连接和防火墙设置
4. 参考完整文档: [README.md](README.md)
